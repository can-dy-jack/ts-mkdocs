# Internationalization (i18n)

ts-mkdocs ships a built-in `i18n` plugin for multi-language documentation sites.

## Directory layout

Organize content under locale directories inside `docs/`:

```
docs/
├── assets/          # shared static files
├── en/
│   ├── index.md
│   └── tutorial/
└── zh/
    ├── index.md
    └── tutorial/
```

Shared assets (images, CSS, JS) stay in `docs/assets/` without a locale prefix.

## Configuration

```yaml
plugins:
  - i18n:
      default_language: en
      languages:
        - locale: en
          name: English
        - locale: zh
          name: 简体中文
      fallback_to_default: true
      redirect_default: true
      nav_translations:
        zh:
          Tutorial: 教程
          Reference: 参考
```

Keep `nav` paths **without** locale prefixes — the plugin prefixes them per build pass:

```yaml
nav:
  - Home: index.md
  - Tutorial: tutorial/index.md
```

## Features

| Feature | Description |
|---------|-------------|
| Language switcher | Header dropdown with per-page alternate links |
| Stay on page | Switching languages keeps the same page path when a translation exists |
| hreflang | `<link rel="alternate" hreflang="…">` tags for SEO |
| Search | Separate Lunr index per locale (`search_index_en.json`, …) |
| Redirect | Optional root `/` redirect to the default locale |

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `default_language` | string | first locale | Default locale code |
| `languages` | array | — | Locale list with `locale`, `name`, optional `build` |
| `fallback_to_default` | boolean | `true` | Link to default locale when a translation is missing |
| `redirect_default` | boolean | `true` | Write root `index.html` redirect |
| `nav_translations` | object | `{}` | Per-locale nav title overrides |

## Incompatibilities

`navigation.instant` conflicts with the language switcher. The build emits a warning when both are enabled.
